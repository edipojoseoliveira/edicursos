Operadores lógicos

Seja bem-vindo(a) a segunda parte da aula 06 do curso de lógica de programação com App Inventor, hoje vamos falar sobre os operadores lógicos E e OU. Os operadores lógicos trabalham em conjunto com os operadores de comparação, e são frequentemente usados nas estruturas condicionais que estudamos nas aulas anteriores.

Sabemos que em uma estrutura condicional nós usamos os operadores de comparação para criar as condições que indicarão se um determinado comando será executado ou não, ao usar uma estrutura condicional pode haver a necessidade de criarmos mais de uma condição para decidir se um determinado comando será executado ou não, para conseguirmos fazer duas ou mais condições juntas nós vamos usar os operadores lógicos E e OU. Vamos ver agora como eles funcionam.

Usando o operador lógico E

Imagine que você criou um aplicativo de agenda, e nele você cadastrou o seguinte compromisso “dia 10 às 15:00 horas eu tenho uma reunião”, nesse compromisso você especificou o dia e a hora. Quando chegar as 15:00 horas do dia 10 o seu aplicativo deve mostrar uma notificação.

Para saber quando é o momento certo de exibir a notificação você pode criar a seguinte estrutura condicional: 

Se o dia for igual a 10 E a hora for igual a 15:00 então
    Exibir uma notificação

Percebeu que no teste da estrutura condicional você precisou fazer duas comparações? Foi necessário comparar o dia e a hora. Nessas situações temos os operadores lógicos para resolver nosso problema, neste exemplo foi usado o operador E.

Usando o operador lógico OU

Suponhamos que você tem que criar um aplicativo para controlar a quantidade de produtos em um estoque de uma loja, o dono da loja falou que a quantidade mínima de produtos é 100 e a quantidade máxima de produtos é 500, seu aplicativo tem que exibir uma notificação para o usuário caso a quantidade de produtos esteja menor que 100 ou maior que 500.

Para desenvolver esse aplicativo você vai precisar verificar se a quantidade de produtos é menor que 100 ou maior que 500 e exibir uma notificação caso esse teste retorne Verdadeiro (True), para isso você pode usar a seguinte estrutura condicional. 

Se a quantidade for menor que 100 OU a quantidade for maior que 500 então
    Exibir uma notificação

Quando usar cada operador lógico

Além de saber como usar os operadores lógicos, é necessário saber quando usar cada um. O operador E é usado quando precisamos que o resultado de todos os testes seja verdadeiro para executar um determinado comando, o operador OU é usado quando precisamos que pelo menos um dos resultados dos testes seja verdadeiro para executar um determinado comando.

Vamos analisar melhor o primeiro exemplo de uso do operador E. 

Se o dia for igual a 10 E a hora for igual a 15:00 então
    Exibir uma notificação

Nós usamos o operador E nesse caso porque precisamos que as duas condições sejam verdadeiras para executar o comando de exibir notificação.

Se o dia for igual a 10 mas a hora não for igual a 15:00 então não pode exibir a notificação, se a hora for igual a 15:00 mas o dia não for igual a 10 também não pode exibir a notificação, a notificação só pode ser exibida quando as duas condições forem verdadeiras.

Vamos analisar melhor o segundo exemplo onde usamos o operador OU. 

Se a quantidade for menor que 100 OU a quantidade for maior que 500 então
    Exibir uma notificação

Nesse exemplo foi usado o operador OU porque não precisamos que as duas condições sejam verdadeiras, se pelo menos uma das condições for verdadeira o comando de exibir uma notificação deve ser executado.

Imagine que a quantidade de produtos é 90, nesse caso a primeira condição é verdadeira, mas a segunda condição é falsa, mesmo apenas uma condição sendo verdadeira o comando de exibir uma notificação deve ser executado.

Se a quantidade fosse 600, apenas a segunda condição seria verdadeira, e a primeira condição seria falsa, mesmo assim a notificação deve ser exibida informando o usuário que a quantidade de produtos está acima do desejado.
