Procedimentos

Seja bem-vindo(a) a nossa oitava aula do curso de Lógica de Programação com App Inventor, nessa aula vamos estudar os procedimentos.

O que são procedimentos?

Um procedimento é um conjunto de comandos que farão uma tarefa específica, usar procedimentos é importante para deixar o código de seus softwares mais organizado, além de que sem usar procedimentos é praticamente impossível criar um aplicativo com várias funcionalidades.

Para você ter uma ideia melhor sobre o que são procedimentos vamos pegar como exemplo o WhatsApp, um aplicativo usado para mandar mensagens de texto, fotos, áudios etc.

O WhatsApp tem várias funcionalidades, entre elas podemos citar: exibir os contatos, enviar mensagens, receber mensagens, entre outras.

Podemos imaginar que no código do WhatsApp existem vários procedimentos como, listar contatos, enviar mensagens e receber mensagens.

Conseguiu entender a ideia? Para cada funcionalidade que você tiver em seu aplicativo é necessário usar um procedimento.

Declarando um procedimento simples

Todo procedimento precisa ter um nome e um conjunto de comandos que irá executar.

As regras para criação do nome de um procedimento são as mesmas da criação de nomes de variáveis.

Parâmetros de um procedimento

Os procedimentos também podem receber parâmetros, eles são informações que passamos para os procedimentos usarem.

Um procedimento pode receber quantos parâmetros você quiser.

Por exemplo, imagine que temos um procedimento que calcula a média de três número, para ele fazer o calcula será necessário informar esses três número, fazemos isso passando-os por parâmetro.

Nem todos os procedimentos precisam de parâmetro, por isso seu uso não é obrigatório.

Imagine que você precisa criar um procedimento para consultar a hora atual, esse procedimento não precisaria receber um valor por parâmetro, pois ele apenas deve consultar a hora atual e retornar esse valor.

Retorno de procedimentos

Um procedimento, após ser executada pode retornar ou não uma resposta.

No exemplo usado anteriormente, o procedimento que calcula a média precisa retornar a média calculada.

O procedimento que verifica a hora atual também necessita retornar um valor, nesse caso a hora atual.

Mas também podemos criar procedimentos que não precisem retornar uma resposta, alguns procedimentos são feitos apenas para executar uma tarefa e não precisam necessariamente retornar um resultado.

Por exemplo, se você criar um procedimento usada para enviar mensagens, ele provavelmente vai precisar receber por parâmetro a mensagem que deve enviar, e seu objetivo é enviar essa mensagem para alguém, no entanto o procedimento não precisa retornar um valor.

Conclusão

Nesse post vimos que usar procedimentos em nossos aplicativos faz com que seus códigos fiquem mais organizados, além disso, com o uso de procedimentos podemos reutilizar alguns códigos.
